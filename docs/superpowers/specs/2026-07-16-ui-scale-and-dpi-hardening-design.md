# UI 缩放档位 + 跨机器渲染加固 设计文档

日期：2026-07-16
状态：已与用户确认

## 背景

1. 用户询问卡片（compact）和胶囊（strip）能否自己拖动大小。这两种形态布局按固定尺寸精调，
   自由拖拽必然变形，因此不开放自由缩放，改为提供整体缩放档位。
2. 另一台机器上的胶囊出现图标歪、"99%" 文字压住进度条等问题。判定根因方向：
   分数 DPI 缩放（125%/150%）下的半像素取整、`transform: scale` 图标补偿的模糊偏移、
   贴死的行高在字体度量差异下溢出。需要做普遍性的防御加固，而不是只修单一症状。

## 范围

- **做**：三档 UI 缩放（compact + strip）；胶囊/卡片的布局弹性与渲染锁定。
- **不做**：compact/strip 自由拖拽缩放；expanded 内容缩放（窗口本身已可自由缩放）；
  右键菜单入口；macOS 验收（共用代码但本轮只在 Windows 验证）。

## A. 固定缩放档位

- 设置项 `uiScale ∈ {1, 1.25, 1.5}`，默认 `1`，localStorage 持久化
  （与玻璃透明度、主题的现有机制一致）。
- 设置页"小组件外观"区域加一组三档分段按钮：100% / 125% / 150%，即时生效。
- 生效机制（两件事同步，保证不变形）：
  1. **窗口**：`windowClient.js` 中 compact 与 strip 的全部尺寸计算
     （`WINDOW_SIZES`、`resizeStripWindow` 动态宽高）统一乘以缩放系数；
  2. **内容**：compact/strip 形态下根元素设 CSS `zoom: <系数>`，
     所有 px 等比放大，布局比例不变。
- expanded 不受影响：进入 expanded 时 `zoom` 恢复 1，窗口走现有自由缩放逻辑。
- 位置记忆逻辑不变；切档时窗口原地重设尺寸。

## B. 兼容加固

1. **布局弹性**：胶囊内 `line-height: 1` 等贴死行高放宽；百分比文字与进度条的
   间距改为不受文字实际渲染高度挤压的布局（文字偏大只占满预留空间，不压进度条）。
2. **去 transform 补偿**：`agent-icon--zcode`（scale 1.3）、`agent-icon--kimi`（scale 1.12）
   改为真实 width/height + 负 margin 保持 16px 占位。transform 缩放是分数 DPI 下
   模糊与偏移的主要来源。
3. **字体与文本锁定**：明确字体回退链；根元素加 `-webkit-text-size-adjust: 100%`，
   降低不同机器字体度量差异对布局的影响。
4. **窗口物理像素对齐**：compact/strip 设置窗口尺寸时按显示器 `scaleFactor`
   取整到物理像素（`PhysicalSize`），消除 125%/150% 下的半像素裁切。

## 验证

- 浏览器演示模式：三档切换，卡片/胶囊（横、竖）不变形。
- `desktop:dev` 实机：Windows 显示缩放调 125% / 150%，检查胶囊横竖两方向、卡片、
  图标清晰度、文字与进度条不重叠；切档后窗口位置不丢。
- 现有测试与 `cargo clippy` / 前端构建不回归（改动以前端为主）。
